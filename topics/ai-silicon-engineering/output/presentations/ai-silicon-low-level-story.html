Chapter 1

What is a bit?

A bit is the smallest useful piece of digital information. Physically, it usually means a signal is being interpreted as low or high, not as some mystical math symbol.

signal level

Digital does not mean abstract

A chip reads voltage ranges and decides whether they count as 0 or 1. That decision is what lets physical circuits behave like logic.

Low

We call the signal 0 when the voltage is below a threshold.

High

We call the signal 1 when the voltage is above another threshold.

Noise margin

The chip has some tolerance so tiny electrical noise does not change the meaning.

The important thing is that a bit is not a little number floating in space. It is an interpreted electrical state.

Binary as voltage

The same wire can mean 0 or 1 depending on whether the signal is near ground or near the supply voltage.

Chapter 2

What is a transistor?

A transistor is a tiny electrical valve. One control signal decides whether a path is blocked or open, and that is the basic trick behind digital chips.

switch

Open path, closed path

Do not memorize the terminal names yet. First learn the behavior: a control input either allows current to pass or blocks it.

Control pin

The tiny input that says “open” or “closed.”

Current path

The left-to-right route electricity can take when the switch is open.

Switching

The useful work is not the still state. It is the change from one state to another.

The transistor is not a symbol on paper. It is a physical gate that turns a path for current on or off.

Closed versus open

Same device, two states. Left panel: blocked. Right panel: flowing.

Chapter 3

How switches become logic gates

When transistors are wired together, they can behave like NOT, AND, OR, NAND, NOR, and the rest of digital logic.

gates

Logic is just rules for outputs

A logic gate takes inputs and computes one output using a fixed rule.

NOTflip 0 to 1 and 1 to 0
AND1 only if both inputs are 1
OR1 if any input is 1
NANDthe inverse of AND
The key point is that gates are not software functions. They are physical arrangements of transistors that enforce a truth table.

Truth table as picture

The same 0/1 rules can be drawn as a tiny map.

Chapter 4

How memory starts: latch and feedback

Logic gates compute, but they do not remember. Feedback turns a circuit into storage. That is the jump from pure combinational logic into state.

memory

Why memory needs feedback

If a circuit can feed its output back into its input, it can keep a value instead of forgetting it the moment the input changes.

Latch

A level-sensitive storage element. It can be open or closed depending on an enable signal.

Feedback

The output is routed back so the circuit can reinforce its own state.

Stable state

Once the circuit settles, it holds 0 or 1 until told to change.

This is the point where “logic” becomes “state.” State is what makes clocks, registers, counters, and processors possible.

Cross-coupled idea

Two gates can watch each other and keep one remembered value alive.

Chapter 4b

SR latch: the first real memory cell

If you see `S` and `R` with outputs `Q` and `Q̅` and no clock, that is a latch. The feedback makes it remember a bit.

latch

What S and R mean

`S` means set the bit to 1. `R` means reset the bit to 0. The latch keeps the result after the input changes away.

S

Set. Push the stored bit toward 1.

R

Reset. Push the stored bit toward 0.

Q / Q̅

One output is the stored bit, the other is its opposite.

This is memory without a clock. The circuit is stable because the outputs feed back into the inputs.

Cross-coupled latch

Two gates watch each other. That feedback is the whole trick.

Chapter 5

Flip-flops and the clock

A flip-flop is memory that only changes at a clock edge. That edge is what gives digital systems their rhythm.

clocked state

Why the clock matters

The clock makes state changes predictable. Instead of changing whenever the input wiggles, the circuit waits for the right moment.

Edge-triggered

The flip-flop samples its input only on a rising or falling clock edge.

Register

A group of flip-flops storing several bits side by side.

State machine

Memory plus rules for how that memory changes over time.

This is the point where a circuit starts behaving like a machine that has a past and a future.
Chapter 6

RTL: describing hardware in code

RTL is not software. It is a structured description of wires, gates, and state that synthesis tools can turn into real circuits.

HDL

What Verilog is actually doing

`assign` describes combinational logic. `always_ff` describes clocked state. Those two ideas are enough to build a lot.

Combinational

Output depends only on the current inputs.

Sequential

Output depends on current inputs plus stored state.

Synthesis

The tool reads the description and builds gates and flip-flops from it.

The big mindset shift is that you are describing a circuit, not writing an ordered list of steps for a CPU to execute.

Code becomes structure

The same idea can be drawn as text and as a circuit.

Chapter 7

FPGA versus ASIC

An FPGA is reconfigurable hardware built from programmable logic blocks. An ASIC is custom silicon built from fixed transistors and gates.

implementation

Same logic, different physical form

An FPGA uses lookup tables and flip-flops to imitate logic. An ASIC builds the logic directly in silicon.

FPGA

Fast to change, slower and less efficient, great for learning and prototyping.

ASIC

Slow and expensive to make, but fast, compact, and power efficient once built.

If you understand LUTs, flip-flops, and gates, you understand why FPGA is a bridge and ASIC is the final physical product.

LUT versus gate

An FPGA can implement logic by looking up outputs in tiny programmable memory. An ASIC wires the answer in directly.

Chapter 8

The physical design flow

Once RTL exists, synthesis maps it to gates, then place and route turns the gates into a physical layout the foundry can fabricate.

RTL to GDSII

From idea to layout

This is the chain: write behavior, verify it, synthesize it, place it, route it, sign it off, and then export GDSII.

RTL

Describe the circuit behavior.

Synthesis

Turn it into gates and flip-flops.

Place & route

Give every cell a physical location and wire it up.

Signoff

Check timing, layout rules, and correctness.

Chapter 9

Why DFT exists

Once chips are manufactured, they need to be tested. DFT makes the circuit controllable and observable enough to find defects economically.

scan chains

Scan chains turn state into a testable shift register

If you can shift values into flip-flops and shift them back out, you can inspect what is otherwise hidden inside the chip.

Controllability

Force internal nodes into a known state.

Observability

Read the state back out after a test cycle.

ATPG

Software generates the patterns that expose faults.

Flip-flops in a chain

This is what “scan” means in plain language.

Chapter 10

Only now: big AI hardware

Once the low-level parts are clear, the larger machines become easier to understand. GPUs, TPUs, and accelerators are just different ways of arranging the same primitives.

AI hardware

Same building blocks, different emphasis

The point is not that GPUs and TPUs are magic. The point is that they optimize different layers of the same stack: gates, registers, data movement, and compiler mapping.

GPU

Flexible parallel machine with lots of control and a strong software stack.

TPU

Specialized matrix machine that relies heavily on compiler control.

Compiler

Turns the model into a schedule that the hardware can actually run.

If the low-level primitives are clear, the big picture stops being mystical. It becomes a design choice.

Bottom to top map

The chain is simple: signal, switch, gate, memory, RTL, synthesis, flow, accelerator.
